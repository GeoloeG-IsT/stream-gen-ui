# Milestone v1.0: City Chatbot Backend PoC

**Status:** ✅ SHIPPED 2026-01-21
**Phases:** 1-3
**Total Plans:** 10

## Overview

Transform the existing frontend streaming UI comparison into a full-stack city chatbot by reorganizing the codebase, building a FastAPI backend with RAG-powered knowledge retrieval, and integrating a ReAct agent that formats responses as rich UI components. The journey moves from structure (Phase 1) to foundation (Phase 2) to intelligence (Phase 3), delivering a working PoC where users ask city questions and receive streaming responses with Contact cards and Calendar events.

## Milestone Summary

**Key Decisions:**
- Mistral LLM: User preference for European data residency
- FastAPI backend: User preference, good SSE support
- LangChain/LangGraph: User preference, ReAct agent support
- Frontend reorganization first: Clean separation before adding backend
- Fictional city data: PoC doesn't need real data
- Hybrid RAG (0.8 semantic, 0.2 BM25): Better recall for meaning-based queries
- Entity markers (:::contact/:::event): Reliable frontend parsing
- Source attribution server-side only: Cleaner frontend display

**Issues Resolved:**
- Streaming implementation uses `.astream()` not `.invoke()`
- CORS configuration explicit (not wildcard)
- Structured output uses entity marker format

**Issues Deferred:**
- reasoning-delta events for displaying agent thinking (not blocking)

**Technical Debt Incurred:**
- TODO: Distinguish reasoning using metadata.langgraph_node (future enhancement)
